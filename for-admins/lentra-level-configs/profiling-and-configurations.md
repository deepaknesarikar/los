# Profiling and Configurations

Profiles are backend configs which can be created on this screen and then used at product level.&#x20;

Example - All the API level configurations like Dedupe / FileX are part of the Profiles screen.&#x20;

![](<../../.gitbook/assets/image (16).png>)

### Create New profile

You can create new profiles with the help of developer.&#x20;

* Config Name - Name of the config
* Config Description - Description for user understanding
* Value - will be integer and will be used by backend for activative different profiles. Always starts from 1
* Value Description - Description of the each profile

![](<../../.gitbook/assets/image (18).png>)

{% hint style="info" %}
Adding new profile at Lentra level, will automatically insert the newly created profile in the all the institutes and product level profiles. &#x20;
{% endhint %}
